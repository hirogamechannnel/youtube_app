# コーディングルール

## 基本方針
- 最小限のコードで要件を満たす
- 冗長な実装を避ける
- 明示的な要求がない限り追加機能は実装しない

## 命名規則
- PEP 8準拠
- 日本語コメント・docstring使用
- snake_case for variables, functions
- PascalCase for classes

## アーキテクチャ
- モジュラー設計維持
- 機能別ディレクトリ構成
- 適切な責任分離

## エラーハンドリング
- 適切な例外処理
- ユーザーフレンドリーなエラーメッセージ
- リトライ機能（必要に応じて）

## 依存関係
- requirements.txtで管理
- 最小限の外部ライブラリ使用
